# Contributing

## Branch and pull request workflow

1. **Create a feature branch** from `main`:

   ```bash
   git checkout main
   git pull origin main
   git checkout -b feature/your-short-description
   ```

2. **Make changes** with focused commits and clear messages.

3. **Open a pull request** into `main` that explains:

   - **What** you changed  
   - **Why** it should be merged (user-facing value, bug fix, or maintenance)  
   - **How to verify** (e.g. `npm run dev`, specific pages or flows)

4. Request review and address feedback before merge.

Keeping PRs small and well-described helps reviewers and keeps history readable.
